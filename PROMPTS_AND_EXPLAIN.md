# Описание и Промпты

## Обоснование выбора технологий (Explain)

- **Next.js** — выбор ничем не обусловлен, просто у меня был настроен пустой проект с линтерами.
- **React Query** — удобно использовать одновременно как стейт-менеджер и делать запросы к API.
- **TailwindCSS** — последние 4 года я писал исключительно на нем и считаю, что на данный момент это лучшее решение для написания стилей.
- **FSD (Feature-Sliced Design)** — часто используемая архитектура, интересно было посмотреть как с ней справится ИИ.

## Нейросети
- для разработки я использовал **Google Antigravity**
- модели - **Gemini 3.1 Pro** и **Claude Opus 6.4**

## Промпты (Prompts)

### 1. Верстка главной страницы
```text
сдлай верстку этой страницы из figma вместо главной страницы. цвета шрифтов и бекграундов положи в переменные tailwindcss. используй fsd архитектуру для компонентов
@link
```

### 2. Интеграция с API (Каталог)
```text
сделай интеграцию с api для страницы catalog используя dummy json ручку products.
https://dummyjson.com/docs/products

в запросе бери только нужные для таблицы данные

в качестве стор менеджера используй react-query, запрос сделай отдельным хуком и вынеси отдельно по fsd дизайну.

предусмотри папку и для других хуков с запросами и общую обработку ошибок для запросов с выводом ошибки в toast. для тостов используй react-toastify

данные должны быть покрыты типами ts.

сразу реализуй пагинацию используя api и вывод кол-ва товаров. на страницу выводи 20 элементов.

также, реализуй прогресс бар при запросе новых страниц
```

### 3. Пагинация и отмена запросов
```text
при переходе по пагинации сделай так, чтобы новая страница проскроливалась вверх и начиналась с начала уже после загрузки новых элементов.

также допили пагинацию для большего количества страниц. добавь кнопки перехода на самую первую и последнюю страницу и стандартное '...' в середине

также сделай request cancellation если ответ не пришел и вызвана новая страница пагинации
```

### 4. Сортировка
```text
расширь запрос к api сортировкой по цене и оценке.
сортировка должна работать при нажатии на соответсвующий тайтл в таблице. у полей, которые можно сортировать добавь рядом подходящую иконку. если сортировка выполняется то выделяй более темным цветом тайтл и соответствующую порядку сортировки иконку
```

### 5. Сохранение состояния в URL и сброс сортировки
```text
сделай так, чтобы выбранная страница пагинации и сортировка хранилась в get запросе и открывала нужные данные при переходе по урлу

рядом с 'все позиции', если включена сортировка, то сделай кнопку сброса сортировки

также сделай refresh кеша по клику на кнопку
```

### 6. Поиск по товарам
```text
сделай поиск используя api products/search?q=
запрос поиска также сохраняй в get запрос.
результат поиска выводи в таблицу. поиск начинай от 3 символов запросе. при отсутсвии элементов в таблицу выводи, что ничего не найдено
```

### 7. Авторизация
```text
сделай  на главной странице авторизацию используя api https://dummyjson.com/docs/auth
хук также вынести отдельно по fsd

для валидации формы используй react-hook-form + zod. поля обязательны для заполнения. при ошибке выводи текст под соотвествующим инпутом красным цветом.
также нужно сделать при ошибке авторизации если указан неверно логин или пароль.

для декода jwt используй jose

при удачном логине должна открываться страница catalog

если нажат чекбокс 'запомнить данные', то сессию сохрани в localstorage, иначе в sessionstorage. настрой роутинг так, что если сессия нет в сторедже то при перезагрузке редирект на логин. если сессия есть и стучимся на главную, то редирект на каталог
```

### 8. Профиль пользователя и логаут
```text
на странице каталога справа от формы поиска товара сделай блок с аватором юзера в кружке.
по клику выводи модальное окно со всей информацией о юзере. exp и iat сделай в читаемом формате 'HH:mm:ss MMMM yyyy, учитывай часовой пояс, используй date-fns.
снизу добавь кнопку logout.

для модального окна используй headless ui, стилизуй окно как форму на странице auth
```

### 9. Добавление товара
```text
при клике на кнопку добавить должно открываться модальное окно с формой.
инпуты - наименование, цена, вендор, артикул
поля обязательны. также под инпутами выводить ошибку, если поле не заполнено

в конце формы кнопка отправить, в хук api
/products/add

после удачного выполнения выводить toast success с текстом, товар успешно добавлен, использовать имеющийся react-toastify

форму стилизовать в стиль проекта. модалку использовать headless ui
```
